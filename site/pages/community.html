Our Pledge

We as members, contributors, and leaders pledge to make participation in our community a harassment-free experience for everyone, regardless of age, body size, visible or invisible disability, ethnicity, sex characteristics, gender identity and expression, level of experience, education, socio-economic status, nationality, personal appearance, race, religion, or sexual identity and orientation.

Our Standards

‚úÖ Examples of Positive Behavior
  • Using welcoming and inclusive language
  • Being respectful of differing viewpoints and experiences
  • Gracefully accepting constructive criticism
  • Focusing on what is best for the community
  • Showing empathy towards other community members
  • Giving and receiving feedback professionally
‚ùå Unacceptable Behavior
  • The use of sexualized language or imagery
  • Trolling, insulting/derogatory comments, and personal attacks
  • Public or private harassment
  • Publishing others' private information without permission
  • Other conduct which could reasonably be considered inappropriate
  • Advocating for or encouraging any of the above behavior

Enforcement Responsibilities

Community leaders are responsible for clarifying and enforcing our standards of acceptable behavior and will take appropriate and fair corrective action in response to any behavior that they deem inappropriate, threatening, offensive, or harmful.

Scope

This Code of Conduct applies within all community spaces, and also applies when an individual is officially representing the community in public spaces. Examples of representing our community include using an official e-mail address, posting via an official social media account, or acting as an appointed representative at an online or offline event.

Getting Started

1
Fork the Repository

Create your own copy of the repository by clicking the "Fork" button on GitHub.

git clone https://github.com/YOUR-USERNAME/Install-and-Learn-DevLangs.git
2
Create a Branch

Create a new branch for your changes with a descriptive name.

git checkout -b feature/add-new-questions
3
Make Your Changes

Add your content, fix bugs, or improve documentation. Follow the existing structure and style.

4
Test Your Changes

Ensure your code works correctly and doesn't introduce errors.

5
Commit & Push

Write clear, descriptive commit messages following conventional commits.

git commit -m "feat: add 5 new Python questions"
git push origin feature/add-new-questions
6
Open a Pull Request

Go to the original repository and open a pull request with a clear description of your changes.

Contribution Guidelines

  • Code Quality: Follow existing code style and conventions
  • Documentation: Update README and relevant docs when adding features
  • Testing: Test all code examples before submitting
  • Comments: Add clear comments explaining complex logic
  • File Organization: Place files in the correct language/topic folders
  • Commit Messages: Use clear, descriptive commit messages
  • Pull Requests: Keep PRs focused on a single feature or fix

Need Help?

If you have questions or need guidance, feel free to:

  • Open an issue with the "question" label
  • Check existing issues and discussions
  • Review the FAQ for common questions
  • Reach out to maintainers through GitHub
